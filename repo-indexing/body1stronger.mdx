# Body1Stronger Components

## Overview

### Purpose

Use this component for stronger body text styling

### When to Use

The Body1Stronger component is designed for maximum emphasis within body text content that requires the strongest visual weight while maintaining the same font size as standard Body1 text. Use Body1Stronger when you need:

- Text that requires the highest level of emphasis or attention within body content
- Critical information, warnings, or alerts that demand immediate attention
- Maximum visual weight for essential statements or key takeaways without changing font size
- Consistent typography that follows the Fluent Design System's text hierarchy with the strongest emphasis level
- Call-to-action statements, important notices, or critical instructions that must stand out
- Error messages, success confirmations, or status updates that require maximum visibility
- Text content that needs to be the focal point within a section while remaining at body text size

### Architecture

**No Coupling** - Body1Stronger is a standalone typography component that doesn't require other components to function. It's part of the broader typography system but operates independently, providing the strongest emphasized body text styling available.

### Component Interdependencies

Body1Stronger has no required dependencies and can be used as a standalone component. However, it's part of a cohesive typography hierarchy that includes related variants for different emphasis levels and font sizes, representing the strongest emphasis level in the Body1 family.

## Components

### Body1Stronger {Standalone Component}

**Purpose:** Provides the strongest emphasized body text styling that adheres to the Fluent Design System's typography standards. This component ensures consistent font sizing, maximum font weight, line height, and spacing for the most critical readable content while maintaining the same font size as regular Body1 and Body1Strong.

**Interface:**

```typescript
interface Body1StrongerProps {
  /**
   * The content to be rendered as strongly emphasized body text
   */
  children?: React.ReactNode;

  /**
   * The HTML element or React component to render
   * @default 'span'
   */
  as?: React.ElementType;

  /**
   * Applies block-level styling (display: block)
   * @default false
   */
  block?: boolean;

  /**
   * Prevents text wrapping (white-space: nowrap)
   * @default false
   */
  nowrap?: boolean;

  /**
   * Applies text truncation with ellipsis on overflow
   * @default false
   */
  truncate?: boolean;

  /**
   * Text alignment
   */
  align?: "start" | "center" | "end" | "justify";

  /**
   * Applies the italic font style to the content
   * @default false
   */
  italic?: boolean;

  /**
   * Applies the strikethrough text decoration to the content
   * @default false
   */
  strikethrough?: boolean;

  /**
   * Applies the underline text decoration to the content
   * @default false
   */
  underline?: boolean;

  /**
   * Wraps the text content on white spaces
   * @default true
   */
  wrap?: boolean;

  /**
   * Additional CSS class names
   */
  className?: string;

  /**
   * Inline styles
   */
  style?: React.CSSProperties;
}
```

**Usage Example:**

```tsx
import { Body1Stronger } from "@fluentui/react-text";

function Example() {
  return (
    <Body1Stronger>
      This is the strongest emphasized body text that commands maximum attention
      while maintaining readability and consistency with the design system.
    </Body1Stronger>
  );
}
```

## Component Relationships

### Typography Hierarchy

Body1Stronger is part of a comprehensive typography system that provides the maximum emphasis level within the Body1 family:

- **Display**: Largest text for major headings
- **LargeTitle**: Large titles for section headers
- **Title1, Title2, Title3**: Hierarchical title sizes
- **Subtitle1, Subtitle2**: Subtitle variations
- **Body1**: Standard body text
- **Body1Strong**: Emphasized body text
- **Body1Stronger**: Strongest emphasized body text (this component)
- **Body2**: Smaller body text for secondary content
- **Caption1, Caption2**: Small text for captions and labels

## Best Practices

### Usage Guidelines

- Use Body1Stronger sparingly for content that requires maximum emphasis while maintaining body text size
- Ideal for critical alerts, error messages, success confirmations, or urgent call-to-action statements
- Combine with semantic HTML elements using the `as` prop for proper accessibility and meaning
- Reserve for the most important information - overuse will diminish its effectiveness and create visual noise
- Perfect for status messages, warnings, critical instructions, or content that users must not miss

### Accessibility

- Use semantic HTML elements (`as="strong"` for strong emphasis, `as="b"` for visual emphasis, `as="p"` for paragraphs)
- Ensure strongly emphasized text has clear meaning and purpose beyond just visual styling
- Test with screen readers to ensure maximum emphasis is properly conveyed to assistive technologies
- Use very sparingly to avoid overwhelming users with reading difficulties or attention disorders
- Maintain sufficient color contrast when customizing text colors, especially for critical information

### Performance

- Body1Stronger components are lightweight and have minimal rendering overhead
- Use consistent typography components rather than custom CSS to maintain design system coherence
- Avoid excessive nesting of text components within each other
- The strongest emphasis styling is optimized for performance across different devices and browsers

### Design Consistency

- Stick to the typography hierarchy provided by the design system
- Use Body1 for standard text, Body1Strong for moderate emphasis, and Body1Stronger for maximum emphasis
- Maintain consistent spacing and alignment with other typography components
- Use the `block` prop for paragraph-style content to ensure proper line spacing
- Avoid using multiple strongly emphasized elements in close proximity

```tsx

  return (
    <div className="alert-container" role="alert">

        {message}
      </Body1Stronger>

      <Body1 as="p" block>
        {details}
      </Body1>
    </div>
  );
}
```
