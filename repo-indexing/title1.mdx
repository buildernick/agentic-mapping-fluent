# Title1 Component

## Overview

### Purpose

Use the Title1 component for primary title text styling in your application's main content areas, landing pages, and primary focal points.

### When to Use

The Title1 component is designed for prominent display text that serves as the primary heading for major content sections. Use Title1 when you need:

- Main page titles and primary headings that establish the primary content hierarchy
- Hero section titles and landing page headlines that capture user attention
- Primary content area headings that organize major application sections
- Display text that needs maximum visual prominence within the typography scale
- Top-level headings in articles, dashboards, and content pages
- Section titles that serve as the primary entry point for user content consumption

### Architecture

**No Coupling** - Title1 is a standalone typography component that doesn't require other components to function. It operates independently within the FluentUI typography hierarchy, providing the largest and most prominent text styling in the system.

### Component Interdependencies

Title1 has no required dependencies and can be used as a standalone component. However, it's part of a cohesive typography hierarchy that includes related components like LargeTitle, Title2, Title3, Subtitle1, Subtitle2, Body1, and other text styling variants for different hierarchy levels and content purposes.

## Components

### Title1 {Standalone Component}

**Purpose:** Provides the primary title text styling with maximum prominence for main headings and display text

**Interface:**

```typescript
interface Title1Props {
  /**
   * Content to display within the title
   */
  children: React.ReactNode;

  /**
   * Controls text alignment
   * @default "start"
   */
  align?: "start" | "center" | "end" | "justify";

  /**
   * Whether the text is rendered as a block element
   * @default false
   */
  block?: boolean;

  /**
   * Whether text should be italicized
   * @default false
   */
  italic?: boolean;

  /**
   * Whether the text should be underlined
   * @default false
   */
  underline?: boolean;

  /**
   * Whether text should be strikethrough
   * @default false
   */
  strikethrough?: boolean;

  /**
   * Controls text size scaling
   * @default 500
   */
  size?: 100 | 200 | 300 | 400 | 500 | 600 | 700 | 800 | 900 | 1000;

  /**
   * Controls font weight
   */
  weight?: "medium" | "regular" | "semibold";

  /**
   * Controls font family
   */
  font?: "base" | "monospace" | "numeric";

  /**
   * Whether text should be truncated with ellipsis
   * @default false
   */
  truncate?: boolean;

  /**
   * Whether text should wrap
   * @default true
   */
  wrap?: boolean;

  /**
   * Custom HTML element to render as
   * @default "h1"
   */
  as?: React.ElementType;
}
```

**Usage Example:**

```tsx
import { Title1 } from "@fluentui/react-components";

function Example() {
  return <Title1>Welcome to Fluent UI</Title1>;
}
```

## Common Patterns

```tsx

  return (
    <div>
      <Title1 as="h1" block>

      </Title1>

        </Subtitle1>

    </div>
  );
```

## Best Practices

### Usage Guidelines

- Use Title1 for the most important headings that establish primary content hierarchy
- Leverage the `as` prop to ensure semantic HTML structure (h1, h2, etc.) for proper accessibility and document outline
- Reserve Title1 for truly prominent display text - use Title2 or Title3 for secondary headings
- Combine with appropriate spacing and layout components to create clear visual hierarchy
- Use consistently across your application to maintain predictable typography patterns

### Accessibility

- Always use appropriate semantic HTML elements (h1, h2, etc.) via the `as` prop for proper document outline
- Ensure heading hierarchy is logical and sequential for screen readers (h1 → h2 → h3, etc.)
- Use Title1 consistently to help users understand content organization and navigation
- Consider contrast ratios when applying custom styling to ensure readability across different themes
- Provide meaningful heading text that describes the content section for assistive technologies

### Performance

- Title1 components are lightweight and optimized for performance with minimal rendering overhead
- Use semantic HTML elements (h1, h2) when possible for better SEO and accessibility
- The truncate prop efficiently handles long text without impacting layout performance

## Component Relationships

Title1 fits within the broader Fluent UI typography system:

**Typography Hierarchy:**

- **LargeTitle**: Even larger headings for major page sections
- **Title1**: Primary title styling (this component)
- **Title2, Title3**: Secondary and tertiary title levels
- **Subtitle1, Subtitle2**: Subtitle variations for supporting hierarchy
- **Body1, Body2**: Body text variations for content

Use Title1 when you need maximum typographic prominence for main headings, ensuring users can quickly identify and understand the primary content structure of your interface.
