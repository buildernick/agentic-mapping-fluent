# Caption2Strong Components

## Overview

### Purpose

Use this component for strong secondary caption text styling

### When to Use

The Caption2Strong component is designed for emphasized secondary caption-level text content that requires more visual weight than standard Caption2 text. Use Caption2Strong when you need:

- Secondary metadata, timestamps, or auxiliary information that needs to be emphasized while maintaining the smallest readable font size
- Important minimal supporting text that provides critical context and requires visual emphasis
- Status indicators, alerts, or system messages in caption size that need to stand out without becoming too prominent
- Emphasized helper text, warnings, or guidance that supports primary content with stronger visual weight
- Key timestamps, version numbers, or copyright text that should be noticeable but still minimal
- Form validation messages or error text that needs emphasis at the smallest text size
- Important footnotes, disclaimers, or supplementary information that users should notice
- Enhanced visibility for critical fine print, legal text, or terms that must be both minimal and emphasized
- Small but important labels for data points, interface elements, or metadata that require user attention

### Architecture

**No Coupling** - Caption2Strong is a standalone typography component that doesn't require other components to function. It's part of the broader typography system but operates independently, providing emphasized styling at the smallest caption text size available in the Fluent Design System hierarchy.

### Component Interdependencies

Caption2Strong has no required dependencies and can be used as a standalone component. However, it's part of a cohesive typography hierarchy that includes related variants for different emphasis levels and font sizes, representing the emphasized version of the smallest text size in the caption family.

## Components

### Caption2Strong {Standalone Component}

**Purpose:** Provides emphasized secondary caption text styling that adheres to the Fluent Design System's typography standards. This component ensures consistent minimal font sizing with stronger font weight, appropriate line height, and spacing for emphasized secondary caption-level content while maintaining excellent readability at the smallest recommended text size.

**Interface:**

```typescript
interface Caption2StrongProps {
  /**
   * The content to be rendered as emphasized secondary caption text
   */
  children?: React.ReactNode;

  /**
   * The HTML element or React component to render
   * @default 'span'
   */
  as?: React.ElementType;

  /**
   * Applies block-level styling (display: block)
   * @default false
   */
  block?: boolean;

  /**
   * Prevents text wrapping (white-space: nowrap)
   * @default false
   */
  nowrap?: boolean;

  /**
   * Applies text truncation with ellipsis on overflow
   * @default false
   */
  truncate?: boolean;

  /**
   * Text alignment
   */
  align?: "start" | "center" | "end" | "justify";

  /**
   * Applies the italic font style to the content
   * @default false
   */
  italic?: boolean;

  /**
   * Applies the strikethrough text decoration to the content
   * @default false
   */
  strikethrough?: boolean;

  /**
   * Applies the underline text decoration to the content
   * @default false
   */
  underline?: boolean;

  /**
   * Wraps the text content on white spaces
   * @default true
   */
  wrap?: boolean;

  /**
   * Additional CSS class names
   */
  className?: string;

  /**
   * Inline styles
   */
  style?: React.CSSProperties;
}
```

**Usage Example:**

```tsx
import { Caption2Strong } from "@fluentui/react-components";

function Example() {
  return <Caption2Strong>Important: Last updated 2 minutes ago</Caption2Strong>;
}
```

## Component Relationships

### Typography Hierarchy

Caption2Strong is part of a comprehensive typography system that provides emphasized styling at the smallest text size in the Fluent Design System hierarchy:

- **Display**: Largest text for major headings
- **LargeTitle**: Large titles for section headers
- **Title1, Title2, Title3**: Hierarchical title sizes
- **Subtitle1, Subtitle2, Subtitle2Stronger**: Subtitle variations with different emphasis levels
- **Body1, Body1Strong, Body1Stronger**: Body text variants with increasing emphasis
- **Body2**: Smaller body text for secondary content
- **Caption1, Caption1Strong, Caption1Stronger**: Standard and emphasized caption text
- **Caption2**: Smallest caption text for minimal information
- **Caption2Strong**: Smallest emphasized caption text for important minimal information (this component)

## Best Practices

### Usage Guidelines

- Use Caption2Strong for the smallest text content that needs emphasis while maintaining readability and accessibility
- Perfect for critical timestamps, important version numbers, emphasized copyright text, or other supplementary information that requires attention
- Combine with semantic HTML elements using the `as` prop for proper accessibility and meaning
- Use sparingly - the emphasis is most effective when used selectively for truly important minimal content
- Ideal for status messages, alerts, important helper text, or critical metadata that provides context while demanding attention
- Ensure the emphasized text truly warrants the visual weight to maintain the effectiveness of the typography hierarchy

### Accessibility

- Ensure excellent color contrast when using Caption2Strong due to its small size, especially when emphasized
- Use semantic HTML elements (time, small, mark, etc.) with the `as` prop for screen reader compatibility
- Consider that very small text may be difficult for users with visual impairments, even when emphasized
- Provide alternative ways to access important information presented in Caption2Strong for accessibility compliance
- The stronger visual weight helps with accessibility by making important minimal content more noticeable

### Performance

- Caption2Strong renders efficiently as it's a lightweight wrapper around the base Text component
- Use Caption2Strong consistently throughout your application to maintain typography performance
- The component inherits all performance optimizations from the underlying FluentUI Text component
- Use emphasis judiciously to avoid performance impact from overuse of styled text components
